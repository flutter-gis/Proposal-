SVG <symbol> + <use> vs inline SVG

500-element icon × 20 instances. Toggle the technique and theme; watch the document-DOM node count.

Document DOM nodes: —  |  Technique: symbol+use  |  Theme: dark

What this proves:

  • In <symbol>+<use> mode the document holds ~520 nodes: one 515-element symbol definition + 20 <use> refs + chrome.
  • In inline mode the same 20 icons cost ~10,300 document nodes (515 × 20). That is the current app's bottleneck.
  • The theme toggle recolors all 20 icons instantly by flipping CSS variables — proving var(--rust-*) resolves through the <use> shadow boundary. No re-render needed.
  • Icons stay vector-crisp at any size (resize the window / zoom to verify).

Compare the node-count number between the two techniques — that delta (~10,300 → ~520) is the ~95% document-DOM reduction driving the recommendation in docs/emoji-rendering-research.md.
